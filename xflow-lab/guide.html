The xFLOW Method

How the LAB works — the loop every tool follows, the conventions that keep flows portable, and the paradigm for shipping them.

01

The paradigm

xFLOW is a visual node/graph flow builder. You lay down steps as nodes, wire them in the order they run, and the graph becomes the artifact — readable, editable, shareable.

Every tool is a self-contained single HTML file with no build step and no dependencies beyond web fonts. That keeps the whole LAB portable: a tool is a file, a file is a link.

DefaultEvery tool ships its own embedded guide — contextual help that lives with the tool. This page is the canonical guide the embedded ones point back to.
02

The loop

1

Build

Drop steps onto the canvas as nodes. Each node holds a prompt or a pipeline action.

2

Wire

Connect outputs to inputs. The wires define run order and what flows downstream.

3

Run

Walk the graph end to end. Each node resolves in sequence along the wire.

4

Publish

Push to GitHub. The client link never changes — the version label tells them it's current.

03

Conventions

One accent per toolEach tool gets a single accent color (LAB = electric cyan). Re-skin by swapping the --accent vars at the top of the file.
Stage discipline“Render” names the finished-image stage only. Sketch/line stages say sketch or line — never render.
Pipeline orderSketch / line → image → animate → upscale → crop. Lock the plate, swap the content.
Files are linksSingle self-contained HTML, deployed to GitHub Pages. Stable URL; the version label moves, the link doesn't.
Guide by defaultEvery tool ships an embedded guide that echoes this source of truth.
04

LOCKS+ tokens

Prompt tokens that pin behavior across a flow. Append the render-bound pair to any prompt headed for a finished image:

HARDLOCK PLOCK

Other LOCKS+ tokens (PLOCK, etc.) carry their own scope — fill this section with the full token table as the set settles.

Edit meThis is a scaffold. Drop in the complete LOCKS+ token list and what each one pins so the embedded tool guides can quote it verbatim.
05

Deploy

The LAB lives in its own folder. Drop xflow-lab/ inside workflow-tools/ and it resolves to:

…/workflow-tools/xflow-lab/

Tool files sit beside index.html in that folder. Existing tools elsewhere in the repo (like the Process Flow Builder) are linked, never moved — the flagship card points up one level with ../ so nothing about the current paradigm changes.
