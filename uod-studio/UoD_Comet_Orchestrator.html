How this works

    
This tool stamps your concept into Tim's Comet automation script. You shape only the concept — steps 1–4 of the script are handled for you.


    
Run order

    
Open ChatGPT in a browser tab → press Copy for Comet → paste into Comet's assistant window → Comet uses browser control to drive the tab through Planner, Prompter, character sheets, and storyboards.


    
Platform & Dopamine Ladder

    
The Platform selector picks the render target — ChatGPT, Grok, or Gemini. Planning always runs in ChatGPT (Planner + Prompter generate the YAML); Grok and Gemini only render the images in a new tab, so be signed into both for those. ChatGPT zips the assets; Grok and Gemini download them loose.

    
Dopamine Ladder adds Tim's story framework to the Planner step and turns on the logline / scene-summary framing. Off by default for a clean run; Tim bakes it into the Grok and Gemini flows, so flip it on for those.


    
Filling the fields

    
primary_genre and subgenre_hint take a single value. Every other concept field is a list — type items comma-separated. The chips under each field confirm what registered; no chip means it didn't parse (usually a missing comma).


    
Run parameters

    
Prompts is how many clips the run makes; Seconds each is how long every clip runs; Beats / prompt is how many scenic beats are packed inside a single clip. Total runtime is Prompts × Seconds each, and a new beat lands every Seconds-each ÷ Beats-per-prompt. The defaults — 4 · 15 · 12 — make a 60-second short of four clips, each paced as twelve beats (~1.25s apart). The ⓘ beside the Run parameters title opens the full breakdown.


    
Ingesting a script

    
Ingest script reads a screenplay and prefills the structural fields for you — concept name, characters (one per line), locations into environment, a scene-by-scene breakdown into detailed_scene_summary, and Prompts set to the scene count. The interpretive calls — genre, tone, visual style, camera — are left for you and The Planner; the parser never guesses them.

    
Best results come from Final Draft .fdx (it's XML, so every line is tagged and the harvest is exact). PDF, DOCX, and plain text also work via heuristics, but a PDF export is lossier — when you can, get the .fdx. No script yet? Open the Creator readme, download the fillable PDF form, fill it in any PDF reader and save, then use Ingest Creator Doc — the tool reads the form fields directly. The readme also carries a printable field guide.

    
To prefill everything — including genre, tone, visual style and camera — the writer adds an ORCHESTRATOR META block anywhere in the script (top is easiest), entered as Action/General lines:

    
=== ORCHESTRATOR META ===
LOGLINE: ...
PRIMARY GENRE: ...
SUBGENRE: ...
TONE/MOOD: ...
VISUAL STYLE: ...
CAMERA: ...
WORLDBUILDING TAGS: ...
NOTABLE VISUAL ELEMENTS: ...
=== END META ===

    
Ingest strips that block out of the scene summary and maps each line to its field. The META block wins for the interpretive fields; the script's own structure still drives characters, locations, scene summary and prompt count.


    
Characters

    
One character per line — a name, optionally with a look description. Each line becomes its own photoreal identity sheet in Step 3, in order. This box auto-fills on ingest: a Creator Doc's Character Specification blocks are assembled into one line each, and a script contributes its cast by name. Leave it blank to keep the generic template; edit the underlying sheet template under the collapsible in section 03.


    
Concepts & library

    
Templates are read-only seeds — clicking one loads an editable copy into the form. Rename it, fill it out, then Save to add it to My concepts.

    
Duplicate clones the current concept under a new name. New starts blank.


    
Saving & loading your work

    
My concepts and any edits live only in this open page — a refresh or closing the tab clears them. To keep your work between sessions, use Export and Import as a save/load pair.

    
Export library bundles every concept — all of My concepts plus the one currently in the form — into a single uod-concepts.json file and downloads it. That file is your saved workspace: keep it anywhere, or commit it to the UoD repo to version your ideas over time.

    
Import reads a uod-concepts.json back in, repopulating My concepts and loading the first one into the form, so you pick up where you left off. In short: Export = Save, Import = Open.


    
Before a run: confirm The Planner, The Prompter and SJinn all exist in the ChatGPT account you point Comet at. If SJinn isn't reachable from the main chat, step 1's premise breaks.

Writer readme — prep & export

    
Format your script so the Orchestrator prefills the whole concept on upload.


    

      
1 · Write in Final Draft, normally

      
Standard screenplay formatting is all the structure the tool needs. Proper scene headings (INT./EXT. LOCATION - TIME), character cues in caps, and action lines let it auto-fill characters, locations, the scene-by-scene summary, and the prompt count.

    


    

      
2 · Paste this block at the very top

      
A screenplay says what happens — never its genre, tone, look, or camera language. Those live only in your head, so they can't be harvested from the scenes. Put them in this ORCHESTRATOR META block at the head of the script: the tool reads it, fills genre / tone / visual style / camera, and strips the block out of the scene summary so it never pollutes your prompts. Leave any line blank to skip it.

      


=== ORCHESTRATOR META ===
LOGLINE: 
PRIMARY GENRE: 
SUBGENRE: 
SECONDARY GENRES: 
TONE/MOOD: 
VISUAL STYLE: 
CAMERA: 
NARRATIVE ARCHETYPE: 
WORLDBUILDING TAGS: 
NOTABLE VISUAL ELEMENTS: 
=== END META ===

        
      

      
Tip: reuse the same block across every episode of a season to lock one house style — only the scene content changes.

    


    

      
3 · Export as Final Draft (.fdx)

      
In Final Draft: File → Save As and choose Final Draft (.fdx). The .fdx is XML, so every line is tagged and the harvest is exact. PDF and DOCX also work but are lossier — send the .fdx whenever you can.

    


    

      
4 · Upload

      
Back here, click ⥥ Ingest script and pick your .fdx. The concept fields fill in — review the chips under each field and you're ready to run.

    

  


  
  

    
    
Creator readme — concept intake

    
For shaping a concept without a screenplay. Fill the sheet, save it, and Ingest it as a Creator doc.


    

      
What this is for

      
When you have a concept, treatment, or teaser idea rather than a finished script, describe it in this intake sheet and the Orchestrator prefills every concept field on upload. Each line maps to one field; clear a line to leave that field blank.

    


    

      
The intake form

      
Download the fillable PDF, open it in any PDF reader, type into the fields, and save. Then upload it with Ingest Creator Doc — the tool reads the form fields directly, so the values land exactly as typed. Nothing to copy, no formatting to get right.

      

        
      

    


    

      

        
Field guide — what goes in each field

        
      

      
New to this? Print this and keep it beside you while filling the form.

      

        
Concept fields

        

          
Concept name
A short title for this piece — e.g. "TESTAMENT — Teaser."

          
Logline
One or two sentences: who it's about, what they want, what's at stake.

          
Scene summary
The beat-by-beat spine, in order. This is what the Planner turns into shots, so walk the action as it happens.

          
Primary genre
The single dominant genre — e.g. science fantasy, thriller, drama.

          
Subgenre
A narrower flavor — e.g. post-apocalyptic mythic, neo-noir.

          
Secondary genres
Other genres in the mix, comma-separated.

          
Tone / Mood
The emotional register as comma-separated adjectives — e.g. reverent, ominous, tragic.

          
Visual style
The look and finish: medium, palette, lensing, lighting — e.g. cinematic, painterly, high-contrast, anamorphic.

          
Camera motion
How the camera moves — e.g. slow push-in, crane reveal, steady tracking, handheld.

          
Subjects
The main people and objects on screen.

          
Environment
Where it takes place — the locations and what they feel like.

          
Narrative archetype
The underlying story shape — e.g. the quest, the call to adventure, rise-and-fall.

          
Worldbuilding tags
Setting keywords — factions, tech, era, places that anchor the world, comma-separated.

          
Notable visual elements
Signature images, motifs, or hero props that must appear.

          
Confidence notes
Anything provisional — placeholders, or what to emphasize / play down.

        

        
Character specification sheet — one block per principal

        

          
Name
The character's name; becomes the identity-sheet title. A block only counts if Name is filled.

          
Age / apparent age
How old they read on screen.

          
Build & height
Physique, stature, body type.

          
Hair & face
Hair color and style, plus key facial features.

          
Wardrobe
Costume, materials, palette, signature pieces.

          
Distinguishing features
Scars, tattoos, marks, props — anything that makes them unmistakable.

        

        
Fill one block per principal character; six blocks are provided, use as many as you need and leave the rest blank. On upload, each filled block is assembled into one character entry automatically and becomes a photoreal identity sheet — nothing to paste.

      

    

  


  
  

    
    
Run parameters — how the numbers work

    
Three numbers describe the shape of the piece. They're passed to The Planner and The Prompter as intent, then the prompts are built to match.


    

      
The three numbers

      

        

          
Prompts
How many video prompts the run produces. One prompt = one generated clip. This is the number of shots in the piece.

          
Seconds each
The length of every clip, in seconds. 15 is a common per-clip ceiling for Veo-style generators, so each prompt is built to fill ~that long.

          
Beats / prompt
How many distinct scenic beats — micro-moments of action or image — are packed inside one clip. Higher means denser, faster-cutting action within the shot; lower means slower and more held.

        

      

    


    

      
How they combine

      

        

          
Total runtime
Prompts × Seconds each. The clips play back to back.

          
Beat cadence (inside a clip)
Seconds each ÷ Beats per prompt — how often a new beat lands.

          
Total beats
Prompts × Beats per prompt — every beat across the whole piece.

        

      

    


    

      
Worked example — the defaults (4 · 15 · 12)

      

        

          
4 prompts × 15 seconds
= 60 seconds of footage — a one-minute short, told in four clips.

          
15 seconds ÷ 12 beats
≈ a new beat every ~1.25 seconds inside each clip — rapid, dopamine-ladder pacing.

          
4 prompts × 12 beats
= 48 beats total across the short.

        

        
So "Beats / prompt" is per clip, not total: 12 means twelve beats packed into each single 15-second prompt — and the cadence is just Seconds each ÷ Beats. Want slower, more cinematic shots? Lower the beats. Want frantic, punchy cutting? Raise them.
